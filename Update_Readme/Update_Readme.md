# 🚀 无人机水华监测系统 —— 升级版方案构想（条件注入 vs 路由）

## 一、背景与对比基准

### 1.1 第一版方案（路由版）

第一版 AI 视觉核心逻辑为 **“天气分类 → 路由选择”**：
1.  使用天气分类模型（ResNet-18）从航拍图像中**猜测**天气（晴天/阴天/雾天）。
2.  根据猜测结果，**从三个专用的 DeepLabV3+ 分割模型中挑选一个**执行语义分割。

**问题**：
- 天气分类可能猜错，一旦猜错，后续分割模型完全错配，误差直接传导，无补救机会。
- 无法利用任何外部结构化数据（如水温、光照、营养盐浓度等生物学/环境数据）。
- 三个天气专用模型各自为政，无法共享跨天气的知识。

### 1.2 升级版方案（条件注入版）

升级版将视觉模块重构为 **“结构化数据条件注入的单一语义分割模型”**：
- 模型本身是一个**条件自适应**的分割网络，同时接收**航拍图像**与**结构化数据**（天气、生物学/环境数据）。
- 不再执行“猜天气 → 切换模型”的流程，而是将天气作为**特征条件之一**，与其他结构化数据一起编码，注入到分割网络中，引导特征提取与像素分类。

## 二、升级版 vs 第一版：核心改进对比

| 对比维度 | 第一版（路由） | 升级版（条件注入） | 改进点 |
| :--- | :--- | :--- | :--- |
| **天气信息获取** | 必须由天气分类模型**猜测** | 优先用户**直接提供**；若缺失，复用天气分类模型**自动补全** | 天气从“路由键”降级为“可补全特征”，消除错配风险 |
| **生物学/环境数据** | **完全无法利用** | 编码为条件特征，与图像特征深度交互（拼接/交叉注意力） | 首次将外部先验知识注入视觉模型 |
| **模型数量** | 4个（3个专用 + 1个天气分类） | 2个（1个条件分割模型 + 天气分类保留作为补全工具） | 由分治架构统一为单一自适应模型 |
| **路由错误传导** | 天气猜错 → 分割模型全错 | 无路由环节；天气即使缺失，也能由补全机制填上，不准仅影响条件信号强度，不会致盲模型 | 鲁棒性本质提升 |
| **数模协同** | 视觉与数模独立，无知识共享 | 同一套结构化数据同时驱动条件分割与机理预测 | 知识同源，系统紧耦合 |
| **跨天气泛化** | 三个模型独立训练，数据割裂 | 单一模型共享所有天气数据，从数据中学习天气不变特征 | 数据利用率更高，泛化性更强 |

## 三、升级版数据流与天气补全机制

```
用户输入：航拍图像 + 结构化数据（天气、水温、光照、营养盐等）
                │
                ▼
    ┌─── 结构化数据中“天气”字段是否有效？ ───┐
    │                                        │
    ▼ 是                                     ▼ 否（用户未提供天气）
【直接使用用户天气】              【调用第一版天气分类模型】
                                 从图像估算天气类别（sunny/cloudy/hazy）
                                 自动填入结构化数据天气字段
    │                                        │
    └────────────────┬───────────────────────┘
                     ▼
          完整的结构化数据 + 图像
                     │
                     ▼
           条件注入语义分割模型
        （DeepLabV3+ 或更强，加入条件编码分支）
                     │
          ┌──────────┴──────────┐
          ▼                     ▼
    分割掩码图              量化结果（面积、覆盖率等）
```

> **关键设计**：第一版的天气分类模型被**完整保留**，但身份从“路由决策者”转变为**数据完整性补全工具**。它仅在用户未提供天气时被触发，输出作为默认填充值，**不参与任何路由决策**。

## 四、其他结构化数据的缺失处理：权威季节默认值

除天气外，用户上传的结构化数据（水温、光照时数、总磷、总氮等）也可能部分缺失。

### 4.1 默认值填充策略

- **按季节**：根据图像拍摄日期自动判定季节（春/夏/秋/冬），填充对应季节的统计典型值。
- **数据来源**：所有默认值均基于**权威机构公开数据**统计而成，确保可信、可查、可答辩：
  - 水温、光照时数：中国气象局国家气候中心发布的各季节/区域平均值。
  - 水体营养盐（总磷、总氮等）：生态环境部《中国生态环境状况公报》中对应区域湖库的典型季节均值。
  - 浮游植物生长相关参数：已发表的藻类生态学文献综述（如微囊藻生长速率与温度关系）。
- **拓展性**：若条件允许，默认值可进一步按**省份/地理分区**细化，以更贴近当地实际。此作为选增强化项，不强求。

### 4.2 填充逻辑

在推理管道中执行：

1.  解析用户上传的结构化数据 JSON，逐字段检查是否存在合法值。
2.  对于缺失的字段，读取图像文件名中的日期，确定季节（也可从 EXIF/GPS 进一步锁定省份）。
3.  从默认值配置库（`params_seasonal_defaults.json`）中查询对应季节（及可能省份）的默认值，填入。
4.  最终构建出**完整无缺**的结构化条件向量，送入分割模型。

## 五、数模预测模块的同步升级（简述）

数模模块同样接收同一套结构化数据（含用默认值补全后的部分）：
- 若有实测环境数据，Logistic 生长方程的参数通过非线性最小二乘法对实测面积序列与实测环境数据进行**联合拟合**，得到个性化参数。
- 若环境数据缺失，则同样使用权威季节默认值，退化为基于默认环境驱动力的机理预测，或进一步退化为纯统计 ARIMA 预测。
- 真正实现 **“同一套先验知识，同时驱动分割与预测”**。

## 六、升级版与第一版的关系：增量增强，而非推倒重来

| 第一版模块 | 升级版中的状态 |
| :--- | :--- |
| 天气分类模型（ResNet-18） | **保留**，降级为天气字段补全工具 |
| 三个天气专用 DeepLabV3+ 模型 | **被替代**，功能由单一条件注入模型吸收；可保留作为对比基线 |
| 统一 DeepLabV3+ 模型（三合一） | **不作为对比对象**，仅在第一版内部用于衬托路由策略的有效性 |
| 推理管道 `predict_pipeline.py` | **重写**，加入结构化数据解析、补全逻辑、条件注入调用 |
| ARIMA / Logistic 预测模型 | **保留并增强**，支持接收真实环境数据拟合，缺失时使用默认值 |

---

## 七、改进点一句话总结

> **从“猜天气、选模型”的被动分治路由，升级为“用天气、缺就补、知识注入”的主动条件融合。外部结构化数据首次同时驱动视觉分割与数模预测，系统从松耦合进化为知识驱动的紧耦合智能体。**
