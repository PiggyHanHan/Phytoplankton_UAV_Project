### 无人机浮游植物识别项目 - 数模时序预测（刘俊辉）专属 README

#### 一、核心定位
作为项目数模预测负责人，我将主导构建一个**可优雅降级的单一生物统计学模型**，并融合生态分区与全国季节知识库，为水华趋势提供高可靠性的预测。不再开发传统双模型，而是聚焦于一个能根据输入数据丰富度自适应调整的智能预测核心。最终负责输出未来数值、水华风险等级，并协同 AI 模块的空间仿真脚本生成预测分布图。

#### 二、绝对清晰的工作边界
✅ **核心负责（优雅降级模型 + 分区建模 + 知识库 + 空间仿真调用）**
1. **分区统计脚本开发**：独立编写 `utils/partition/ecological_zone_partition.py`，从吴天宇交付的掩码图中按生态规则（岸水区、浅水区、深水区等）自动提取各分区的浮游植物面积时序，为精细化建模打基础。
2. **全国知识库构建**：通过查阅文献与权威公报，制作 `models/forecast/region_seasonal_defaults.json`，为全国各主要省份/地区提供春、夏、秋、冬四季的生物学默认参数（水温、光照、最大生长速率 r_max、环境容纳量 K 等），并明确标注数据来源。
3. **单一模型搭建**：开发 `models/forecast/bio_stat_model.py`，核心为 Logistic 生长方程骨架，融入以下两级自适应机制：
   - **数据富集时**：接收用户提供的实测水温、光照等环境变量，通过非线性最小二乘法精确拟合模型参数，实现机理驱动的精准预测。
   - **数据缺失时**：自动根据监测点的 GPS 定位，从知识库中匹配对应省份与季节的默认参数，结合历史面积统计，退化为基于科学先验的稳健统计预测。模型可量化输出此次预测对先验知识的依赖程度。
4. **分区预测与集成**：对每个生态分区独立运行模型，得到各分区未来的面积演化序列。最终将所有分区的预测结果按空间位置科学加总，输出形式上与原总图统计口径完全一致的总体预测值。
5. **退化性分析**：编写 `degradation_analysis.py`，横向对比模型在“全实测数据”、“部分默认参数”、“完全默认参数”三种场景下的预测表现，产出退化性分析报告，作为答辩核心亮点。
6. **预测分布图生成**：
   - 调用吴天宇交付的 `utils/simulation/extract_growth_pattern.py`，从连续实测掩码中提取空间扩张规律。
   - 将未来面积预测值（整体或分区）及生长规则传入 `utils/simulation/generate_prediction_mask.py`，生成逼真的未来掩码图，存入 `outputs/masks/`。
7. **数据交付**：预测数值 JSON 写入 `outputs/quantifications/`，确保格式与吴天宇输出的实测数据完全对齐，供展示组直接读取。

❌ **绝不涉及**
- 不参与无人机图像预处理、标注。
- 不参与 AI 模型训练、推理。
- 不开发前端展示界面。

#### 三、模型原理简述
- **核心方程**：dN/dt = r * N * (1 - N/K)
- **参数动态化**：r 和 K 均为环境变量（水温 T、光照 L 等）的函数，形式可为 r = r_max * f(T) * g(L)。
- **降级逻辑**：当缺乏 T, L 实测值时，T, L 从知识库按 GPS 定位的季节默认值中读取，r_max 和 K 使用对应地区季节默认先验，并结合历史面积数据通过贝叶斯或加权方式进行约束。

#### 四、与 AI 模块的协作接口
1. **读取**：`outputs/quantifications/` 中的时序面积 JSON，以及 `outputs/masks/` 中的对应日期的掩码图。
2. **调用吴天宇工具**：
   - `extract_growth_pattern.py`：输入两个连续实测掩码路径，输出生长规则 JSON。
   - `generate_prediction_mask.py`：传入基准掩码、目标面积变化率及生长规则，输出预测掩码图。
3. **上图下文**：我产出的预测掩码与量化数据均存入 `outputs/`，乔梓阁的展示端直接读取。

3. `models/forecast/degradation_analysis.py` 及退化性分析报告。
